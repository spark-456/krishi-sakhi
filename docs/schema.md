## Conventions Applied Throughout

- All primary keys are `uuid` generated by `gen_random_uuid()`
- All tables have `created_at timestamptz NOT NULL DEFAULT now()`
- Tables that track mutable state also have `updated_at timestamptz NOT NULL DEFAULT now()` — managed by a trigger
- Foreign keys are named `{referenced_table_singular}_id`
- `farmer_id` appears in nearly every table as a denormalised FK for fast context assembly and RLS enforcement — do not remove it even where it could be derived through joins
- RLS is **enabled on every table** — no exceptions
- Enums are stored as `text` with a CHECK constraint rather than PostgreSQL ENUM types, for easier migration management
- `jsonb` is used where structure may evolve (context blocks, recommendation scores)
- Monetary values are stored in Indian Rupees as `numeric` — no currency conversion

## Section 3 — Crop Management Domain

### `crop_records`

Tracks what is being grown on a specific farm in a specific season. Only one record per farm should have `status = 'active'` at any time — enforced by a partial unique index. The `farmer_id` column is denormalised here intentionally: the FastAPI context assembler must retrieve the active crop record in a single lookup without an additional join back through `farms`.

| Column | Type | Constraints | Notes |
|---|---|---|---|
| id | uuid | PK, DEFAULT gen_random_uuid() | |
| farm_id | uuid | NOT NULL, FK → farms.id ON DELETE RESTRICT | Restrict prevents deleting a farm with crop history |
| farmer_id | uuid | NOT NULL, FK → farmers.id | Denormalised — do not remove |
| crop_name | text | NOT NULL | Should match ref_crops.crop_name_en where possible |
| season | text | NOT NULL, CHECK IN ('kharif','rabi','zaid') | |
| sowing_date | date | | |
| expected_harvest_date | date | | |
| actual_harvest_date | date | | Null until status becomes 'harvested' |
| growth_stage | text | CHECK IN ('germination','vegetative','flowering','maturity','post-harvest') | Updated manually or by advisory context |
| status | text | NOT NULL, DEFAULT 'active', CHECK IN ('active','harvested','abandoned') | |
| created_at | timestamptz | NOT NULL, DEFAULT now() | |
| updated_at | timestamptz | NOT NULL, DEFAULT now() | |

**Indexes:**
- Primary key on `id`
- Index on `farmer_id`
- Index on `farm_id`
- Partial unique index on `(farm_id)` WHERE `status = 'active'` — enforces one active crop per farm

**RLS Policies:**
- `SELECT`: `auth.uid() = farmer_id`
- `INSERT`: `auth.uid() = farmer_id`
- `UPDATE`: `auth.uid() = farmer_id`
- `DELETE`: Disabled — crop records are permanent history

### `advisory_messages`

The most important table in the system. Every farmer query and every system response is recorded here with full audit data. This table grows indefinitely and is the primary source for future model evaluation, retraining signal, and outcome measurement.

`context_block_sent` stores the exact JSON payload sent to Dify for each turn — this is essential for reproducibility and audit. `retrieved_chunk_ids` records which knowledge base chunks were used, linking the response to its sources.

| Column | Type | Constraints | Notes |
|---|---|---|---|
| id | uuid | PK, DEFAULT gen_random_uuid() | |
| session_id | uuid | NOT NULL, FK → advisory_sessions.id | |
| farmer_id | uuid | NOT NULL, FK → farmers.id | Denormalised |
| input_channel | text | NOT NULL, CHECK IN ('text','voice','image') | |
| farmer_input_text | text | NOT NULL | Typed text or Whisper transcription. Never raw audio |
| whisper_confidence | numeric | CHECK (whisper_confidence BETWEEN 0 AND 1) | Null when input_channel = 'text' |
| context_block_sent | jsonb | NOT NULL | Full assembled context payload sent to Dify. Must always be populated |
| retrieved_chunk_ids | text[] | | Dify knowledge base chunk IDs used in generation. Empty array if RAG retrieval returned nothing |
| response_text | text | NOT NULL | Final advisory response returned to farmer |
| was_deferred_to_kvk | boolean | NOT NULL, DEFAULT false | True when safety guardrail triggered KVK deferral |
| response_latency_ms | integer | | End-to-end response time for this turn in milliseconds |
| created_at | timestamptz | NOT NULL, DEFAULT now() | |

**Indexes:**
- Primary key on `id`
- Index on `farmer_id`
- Index on `session_id`

**RLS Policies:**
- `SELECT`: `auth.uid() = farmer_id`

## Section 6 — ML Outputs Domain

### `soil_scans`

Created when a farmer uploads a soil image and YOLOv8n runs classification. The raw image is stored in the `soil-images` S3 bucket; this table stores the structured result and a reference to the file. `manually_corrected` and `corrected_soil_class` support a future agronomist review workflow where misclassifications can be labelled for retraining.

| Column | Type | Constraints | Notes |
|---|---|---|---|
| id | uuid | PK, DEFAULT gen_random_uuid() | |
| farmer_id | uuid | NOT NULL, FK → farmers.id | |
| farm_id | uuid | NOT NULL, FK → farms.id | |
| advisory_message_id | uuid | FK → advisory_messages.id | The message turn that triggered this scan. Nullable if scan was initiated outside advisory flow |
| storage_path | text | NOT NULL | Full path in soil-images bucket: `{farmer_id}/{farm_id}/{timestamp}.jpg` |
| predicted_soil_class | text | NOT NULL, CHECK IN ('clay','loam','sandy','red','black','alluvial') | |
| confidence_score | numeric | NOT NULL, CHECK (confidence_score BETWEEN 0 AND 1) | YOLOv8n softmax output |
| manually_corrected | boolean | NOT NULL, DEFAULT false | |
| corrected_soil_class | text | CHECK IN ('clay','loam','sandy','red','black','alluvial') | Null unless manually_corrected = true |
| created_at | timestamptz | NOT NULL, DEFAULT now() | |

**Post-insert trigger:** On INSERT, update `farms.soil_type` with `predicted_soil_class` for the associated `farm_id`.

**Indexes:**
- Primary key on `id`
- Index on `farmer_id`
- Index on `farm_id`
- Index on `manually_corrected` — for retraining dataset extraction

**RLS Policies:**
- `SELECT`: `auth.uid() = farmer_id`
- `INSERT`: Service role only (written by ML microservice via backend)
- `UPDATE`: Service role only (for manual correction workflow)
- `DELETE`: Disabled — training data must not be deleted

## Section 8 — Triggers

### `updated_at` auto-update trigger
Applied to: `farmers`, `crop_records`
```
Fires: BEFORE UPDATE
Action: SET updated_at = now()
```

### `advisory_sessions_total_turns` increment trigger
Applied to: `advisory_messages`
```
Fires: AFTER INSERT
Action: UPDATE advisory_sessions SET total_turns = total_turns + 1
        WHERE id = NEW.session_id
```

### `soil_scan_updates_farm` trigger
Applied to: `soil_scans`
```
Fires: AFTER INSERT
Action: UPDATE farms SET soil_type = NEW.predicted_soil_class
        WHERE id = NEW.farm_id
Condition: Only fires when manually_corrected = false
```

## Section 12 — Context Assembly Query

This is the query the FastAPI context assembler runs on every advisory request. It must remain performant. Do not add joins or subqueries that would slow this path.

```
Given: farmer_id (from JWT)

Reads:
  1. farmers WHERE id = farmer_id
     → full_name, district, preferred_language

  2. farms WHERE farmer_id = farmer_id (most recently updated)
     → farm_name, area_acres, soil_type, irrigation_type

  3. crop_records WHERE farmer_id = farmer_id AND status = 'active'
     → crop_name, season, growth_stage, sowing_date, expected_harvest_date

  4. expense_logs WHERE farmer_id = farmer_id
     AND crop_record_id = (active crop record id)
     AND expense_date >= now() - INTERVAL '30 days'
     GROUP BY category
     → SUM(amount_inr) per category

  5. Open-Meteo API (external, not DB)
     → current temperature, humidity, rainfall, forecast

Assembled into context_block_sent (jsonb):
  {
    "farmer": { name, district, language },
    "farm": { name, area_acres, soil_type, irrigation_type },
    "active_crop": { name, season, growth_stage, sowing_date },
    "recent_expenses": { seeds: X, fertilizer: Y, ... },
    "weather": { temp, humidity, rainfall, forecast },
    "ml_outputs": { ... if image was uploaded this turn }
  }
```
